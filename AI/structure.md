# AI 친화적인 백엔드 구조

AI 친화적인 백엔드 구조는 
전통적인 비즈니스 로직 처리 방식을 넘어, AI 모델을 효율적으로 통합하고 실시간으로 데이터를 처리하며 확장성을 확보하는 데 중점을 둔다.

1. 핵심 아키텍처 패턴: MSA + Headless 
- 마이크로서비스 아키텍처(MSA) : AI 서비스, 데이터 처리, 일반 비즈니스 로직을 독립적인 서비스로 분리하여 특정 AI 모델의 부하가 전체 시스템에 영향을 주지 않도록 합니다.
- 헤드리스(Headless) API 중심 구조 : 프론트엔드와 분리된 API 중심 설계로, AI 가 상품, 주문, 고객 데이터에 쉽게 접근하여 예측 및 개인화 기능을 수행할 수 있도록 한다. 


2. 데이터 및 추론 파이프라인(RAG 및 오케스트레이션)

* RAG (Retrieval-Augmented Generation) : LLM 이 모르는 걸 외부 지식에서 찾아서 답하게 만드는 구조 
LLM 단독 = 뇌
RAG = 뇌 + 검색엔진 + 메모리 
LLM 은 최신 정보를 모르고, 회사 내부 데이터를 모르고, 길게 기억 못함, 환각(Hallucination)
- RAQ(검색 증강 생성 기반) : LLM 이 외부 데이터를 활용하여 할루시네이션(환각)을 줄이고 정확한 응답을 하도록, 데이터 검색과 모델 추론을 분리한 RAQ 아키텍처 채택 
- 오케스트레이션 레이어 : LangChain 등과 같이 사용자 요청, 데이터 검색, 모델 호출(Open AI, Anthropic 등), 결과 후처리를 조율하는 중간 계층을 둡니다. 
- 비동기 처리: AI 모델 호출은 시간이 걸리는 작업이므로, 비동기 메시지 큐(Kafka, RabbitMQ)를 사용하여 백엔드-AI 모델 간 상호작용을 처리하여 응답성을 높입니다. 


3. 데이터 저장 및 관리 (Vector DB)

벡터 데이터베이스(Vector DB): 사용자 컨텍스트나 문서 데이터를 수치화하여 저장하기 위해 PGVector, Pinecone, Milvus 등 벡터 검색이 가능한 데이터베이스를 필수적으로 포함한다.
