# 현재 구조
SSE 스트림 구조로 프론트쪽에서 업로드 하는 과정을 UI 적으로 표현할 수 있다.  
대용량의 파일을 업로드할 때는 청크로 나눠서 처리한다. 

청크로 나눠서 서버로 전송하는 것은 파일 자체를 서버에 전송할 때 대용량 파일이기 때문에 
전송 중에 네트워크 문제로 끊길 경우를 대비하기 위함이다. (대용량 파일을 안정적으로 서버에 전송)

동작:

- 클라이언트가 파일을 여러 청크로 나눠 전송
- 서버는 각 청크를 받으며 임시 저장
- 모든 청크가 도착하면 최종 파일 합치기
- 네트워크 문제로 업로드가 끊겨도 일부 청크만 재전송 가능
- 서버 메모리 부담 감소
- 대용량 파일 업로드 가능



SSE Server sent event (text/event-stream)
현재 구조처럼 SSE 스트림 구조로 개발한 이유는 파일을 서버에 전송이 아닌 , 
전송 후에 서버에서 처리하는 과정을 프론트쪽에 UI 적으로 표현하기 위함이다. 


특징:
방향: 서버 → 클라이언트 (단방향)
동작:
- 서버가 연결을 유지하며 상태, 진행률, 알림 등 이벤트 전송
- 클라이언트는 EventSource 등을 이용해 실시간 수신

장점:

- 업로드 진행 상태, 로그, 알림 등을 실시간으로 보여줄 수 있음
- 브라우저/클라이언트에서 쉽게 처리 가능
