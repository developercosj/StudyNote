1. 운영체제
    - 하드웨어 계층과 다른 소프트웨어 계층 연결하는 소프트웨어 계층
    - 컴퓨터 시스템의 자원 관리
    - 사용자가 컴퓨터를 사용할 수 있는 환경 제공
    - 처리능력, 반환시간, 사용 가능도(얼마나 빨리 제공), 신뢰도
2. CPU
    - 컴퓨터의 뇌 역할
    - 프로그램을 실행하는데 필요한 연산 처리 수행
    - CPU 는 RAM 까지만 도달가능하기 때문에 보조기억장치의 대상을 연산하려면 그 대상을 RAM 에 적재필요 즉 CPU는 주기억장치에 있는 것만 처리
   - MMU
        - CPU 안에 위치
        - CPU 에서 MMU를 거쳐 논리주소에 해당하는 물리주소값 얻음 ( 보호해야 하는 메모리 영역에 대한 접근 제한하여 메모리 보호)
    - CPU는 MMU에서 논리주소를 가지고 메모리의 페이지 테이블에 접근하여 물리주소를 획득한 후 메모리의 물리주소에 접근(2번 접근)
    - TLB : CPU가 자주 요청이 들어오는 논리주소와 물리주소의 매핑정보 저장하는 캐시, MMU안에 존재
    - CPU는 주기억장치(RAM에 있는 메모리)까지만 도달
    - CPU가 처리할 대상은 주기억장치에 있어야함
3. 메모리
    - 데이터를 저장하기 위한 기억장치
    - 휘발성 메모리 : 주기억장치(메인 메모리, RAM)
    - 비휘발성 메모리 : 보조기억장치(SSD, HDD 등)
    - 논리메모리와 물리메모리
        - CPU가 프로세스를 처리할 때 보는 주소값 : 논리 메모리 주소
        - 실제 메모리에서 사용되는 주소값 : 물리주소
        - 논리 메모리와 물리메모리 주소값이 다르므로 변환해 주는 하드웨어 장치 : 메모리 관리 장치(Memory management Unit)
    - 가상 메모리
        - 프로그램 실행시 OS는 디스크에 저장된 데이터를 메모리에 로드하며 메모리 한계를 극복하게 위해 가상메모리 필요
        - 가상메모리는 프로세스의 일부만 메모리에 로드하고, 나머지는 디스크의 스왑영역에 둔 상태로 프로세스를 실행하는 방식
    - 요구 페이징
        - 프로세스에서 필요한 페이지만 메모리(RAM)에 로드하는 방식
        - 페이지 폴트 : 프로그램 실행 시 물리 메모리에 필요한 페이지가 없을때 발생
        - 페이지 폴트 발생시 디스크에서 필요한 페이지를 swap in
        - 메모리에 영역이 찼을때 필요없는 메모리를 스왑영역에 보내는 것을 swap out
    - 페이지 테이블 : 논리주소와 물리주소의 매핑정보가 있는 테이블 (MMU 가 접근함)
