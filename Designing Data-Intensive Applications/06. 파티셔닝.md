# 파티셔닝

파티션을 나눌 때는 보통 각 데이터 단위(레코드, 로우, 문서)가 하나의 파티션에 속하게 한다. 
확장성 : 데이터 파티셔닝의 주된 이유 
단일 파티션에 실행되는 질의 : 각 노드에서 자신의 파티션에 해당하는 질의를 독립적으로 실행할 수 있으므로 노드를 추가함으로써 질의 처리량을 늘릴 수 있다. 



데이터 색인과 파티셔닝의 상호작용 재균형화 



# 파티셔닝과 복제 

보통 복제와 파티셔닝을 함께 적용해 각 파티션의 복사본을 여러 노드에 저장한다. 
한 노드에 여러 파티션을 저장할 수 있다. 
리더 팔로워 복제 모델을 사용하는 것도 가능 -> 각 파티션의 리더는 하나의 노드에 할당, 팔로워들은 다른 노드에 할당 
파티셔닝의 목적 : 데이터와 질의 부하를 노드 사이에 고르게 분산시키는 것 
쏠림이 있으면 파티셔닝의 효과가 매우 떨어진다. 
핫스팟 : 불균형하게 부하가 높은 파티션 
-> 핫스팟을 회피하는 가장 단순한 방법은 레코드를 할당할 노드를 무작위로 선택하는 것 -> 데이터가 노드들 사이에 매우 고르게 분산되지만 어떤 레코드를 읽으려고 할때 해당 레코드가 어느 노드에 저장됐는지 알 수 없음 모든 노드에서 병렬적으로 질의를 실행해야 한다. 

# 키 범위 기준 파티셔닝 

각 파티션에 연속된 범위 (어떤 최솟값에서 최댓값까지)의 키를 할당하는 것 
데이터를 고르게 분산시키려면 파티션 경계를 데이터에 맞춰 조정해야 한다.
