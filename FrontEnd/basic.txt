## 프론트앤드 React 프로젝트 구조

- App.tsx 가 하는 역할
    - 전역 네비게이션
    - 페이지 컨테이너
    - 라우터 Provider
    - 전역 Alert/Toast
    - 전역 상태 초기화
    - 전체 공통 레이아웃

*개별 페이지 내용은 App.tsx 가 아니라 따로 관리한다.


## 전체 구조 정리
- App.tsx : 서버 전체 골격 (전역 미들웨어)
- pages : 라우트 핸들러
- componets : 부분 뷰
- api/ : axios 기반 서비스 코드
- stores : 전역 상태(store)
- hooks : 재사용 가능한 비즈니스 로직(helper)




src/
  api/               ← axios, API 연결
  components/        ← 버튼, 카드, 레이아웃 같은 재사용 컴포넌트
  pages/             ← 페이지 단위 (Home, Login, About)
  routes/            ← 라우터 설정
  stores/            ← zustand (상태 관리)
  hooks/             ← 재사용 가능한 custom hooks
  utils/             ← 유틸 함수
  assets/            ← 이미지/SVG
  App.tsx            ← 앱 전체 Layout
  main.tsx           ← 렌더링 시작점




  ## 화면 흐름


  main.tsx
     ↓
  App.tsx    ← ✨ 전체 틀 (레이아웃 + 라우터)
     ↓
  Router (routes/index.tsx)
     ↓
  pages/ (Home.tsx, Login.tsx, ...)
     ↓
  components/ (Button.tsx, Header.tsx, ...)


## Page import & 라우팅에 대해

- 일반적으로 React Router 를 사용할 때, 페이지 컴포넌트는 필요한 곳에서 import 해서 Route 로 매핑
- 보통은 최상위 라우팅 구성을  App.tsx 에 둔다.
- 페이지가 많아지면 routes 폴더로 분리하거나, 중첩 라우트/lazy 로딩을 사용해 파일을 나눈다.
- 루트에서 라우터/프로바이더 감싸기 - main.tsx
- 페이지가 많아질 때,  App.tsx  가 너무 커지지 않게 하는 방법
    - 중첩 라우팅 : 상위 레이아웃(헤더,사이드바)과 페이지 영역을 분리
    - lazy 로딩 + 코드 스플리팅 : 페이지를 동적으로 import 해서 초기 번들을 줄임
    - routes 




async
- 비동기 작업을 깔끔하게 처리하려고 붙이는 것
- 로그인/회원가입은 보통 서버 API 호출이라 시간이 걸리기 때문에 async 를 붙여서 함수 안에서 await 로 응답 기다리고, 에러도 try/catch 로 처리
