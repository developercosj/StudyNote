
1. kotlin("plugin.spring"), 

Spring AOP 
kotlin class 는 모두 final 이며 kotlin("plugin.spring") 플러그인을 추가하여 컴파일 시 class 를 open 클래스로 변환해준다.
(JVM 은 open 클래스를 봄)
플러그인이 없으면 final class, final fun 이기 때문에 프록시 생성 불가 

Kotlin 은 Java 와 달리 클래스가 기본적으로 final 이라 @Transactional, @Cacheable 같은 AOP 기반 어노테이션이 붙어 있으면 open 이 없으면 런타임 에러가 난다. 

internal 이 붙으면 모듈 내부에서만 쓰도록 의도적으로 접근 제한하는 것이다.

2. kotlin("plugin.jpa")

- JPA 관련 어노테이션이 붙으면 자동으로 open 처리 

JPA는 Lazy Loading 을 할 때 프록시 객체를 사용한다. (상속이 가능해야 Lazy Loading이 동작합니다.)
    동작 방식
            1. 실제 엔티티 대신
           2. 상속 기반 프록시 클래스를 만들어서
           3. 나중에 DB 조회



## 프록시 생성 과정
```
1. Spring 컨테이너 시작
   ↓
2. @Service 클래스 스캔
   ↓
3. @Transactional 등의 어노테이션 발견
   ↓
4. CGLIB으로 프록시 클래스 생성 (상속 필요!)
   ↓
5. 프록시 객체를 빈으로 등록
   ↓
6. 다른 클래스에 주입할 때 프록시 주입


@Transactional이 가장 흔하게 사용되지만, Spring AOP를 사용하는 모든 기능에서 프록시가 필요합니다!
