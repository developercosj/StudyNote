Git Token 생성 종류 

1. Generate new token Fine-grained, repo-scoped 
- 권한을 범위(scope) 단위로 준다. 
- 한번 권한을 주면 계정 전체 repo 에 적용된다. 
- 세부 repo 선택 불가 
- 단순하고 호환성 좋다. 
- 특정 레포만 지정 가능 
- -> 모든 private repo 읽기/쓰기 가능하다. 


2. Generate new token (classic) For general use 
- 최근 Github 가 권장하는 방식 
- 특정 Repository 만 선택 가능 
- 읽기/쓰기 권한을 기능 단위로 세분화한다.
- 조직 정책과 연동 가능하다
- 보안이 훨씬 강하다.
