
확장성 
- 시스템 용량을 늘려 성능 개선, 수요 처리 

탄력성 
- 확장성이 뛰어난 시스템을 동적으로 확장 
- 자동으로 스케일 업과 스케일 다운 수행 


가용성
- IT 서비스 또는 컴포넌트가 작동 가능한 상태에 있는 시간의 비율 


신뢰성
- 시스템이 지정된 간격 동안 의도한 기능을 수행 



다중 계층 : n-tier 
다중 계층 아키텍처에서는 분산 시스템으로 작업할 때 계층을 분리하고 계층 내에서 자원을 공유하는 방식을 고려해야 한다. 

비공유 아키텍처 : 다른 노드들이 해당 노드 또는 서로 간에 메모리, 디스크, CPU 등의 자원을 공유하지 않는다. 




모놀리스

- 가능한 많은 것을 하나에 포함 
- 기술결합 : 아키텍처 계층
- 도메인 결합 : 도메인이 서로 결합하는 방식 
- 모놀리스의 강한 결합은 컴포넌트의 모듈화가 부족하다는 것을 의미 



데이터 아키텍처에 관한 고려 사항 

- 모놀리식하고 강하게 결합된 데이터 아키텍처 


사용자 접근 : 싱글 vs 멀티테넌트 



- 조직 데이터 웨어하우스 아키텍처 


1. 운영 데이터베이스(온라인 트랜잭션 처리 OLTP) 에서 온라인 분석 처리(OLAP)를 분리 
2. 데이터 중앙 집중화 및 구성
