End-to-end tests
    최종 사용자의 흐름에 대한 테스트
    요청, 응답 테스트
Integration tests
    코드의 주요 흐름들을 통합적 테스트
    주요 외부 의존성 테스트 (DB 등)
Unit tests
    도메인 모델과 비즈니스 로직 테스트




- 테스트 코드는 당시 기능 테스트가 아닌 서비스 지속 가능성을 위한 테스트
- 테스트가 문서의 역할을 한다면?
- 좋은 코드는 테스트하기 좋다.
- 테스트 자동화

- DAMP (Descriptive and Meaningful Phrases) 설명적인 구문을 사용하여 코드를 작성하자
