- 자금 숏티지 : 앞으로 돈이 부족해질 것 같은 상황 
ERP 에 이런 데이터가 있다. : 매출채권 회수 예정일 (언제 돈이 들어올지), 
매입채무 지급 예정일(언제 돈이 나갈지), 
급여, 세금 등 고정 지출 일정이 있어서 이것들을 날짜별로 쭉 나열하면 자금 부족 시점과 규모를 미리 볼 수 있음 
- 조달 기안 : 숏티지가 예측됐을 때 부족한 자금을 어떻게 메울지 방법을 결정하고 결재를 받는 문서 
-> 조달방법으로는 은행 단기 대출 실행, 당좌촤월 한도 사용, 계얄사 간 자금 이체, 매출채권 조기 회수 독촉 같은 방법이 있다.
이중 어떤 방법을 쓸지, 금액은 얼마인지, 기간은 언제까지인지를 담아서 CFO 나 재무팀장에게 결재를 올리는 것이 조달 기안 
- 예측 기간을 단기(7일), 중기(30일), 장기(90일) 로 나눠서 보여주면 실무 활용도가 높아진다. 


미지급비용 · 선급비용 결산 수정분개 : 결산 시점에 발생했지만 아직 처리 안 된 비용/수익을 올바른 기간에 맞게 조정하는 작업


- 미지급비용 : 비용은 발생했지만 아직 지급하지 않은 것
- 선급비용 : 미리 지급(선납)했지만 아직 비용이 발생하지 않은 것
- 미수수익 : 수익은 발생했지만 아직 받지 못한 것
- 선수수익 :  미리 받았지만 아직 수익이 발생하지 않은 것



연결 결산 시 내부거래 자동 제거 및 연결 조정 분개 생성의미 
: 그룹 전체를 "하나의 회사"처럼 재무제표를 만들 때, 계열사끼리 주고받은 거래를 지워버리는 작업 
필요성
예를 들어 A그룹에 모회사 A와 자회사 B가 있다고 하면:
A (모회사)  →  B (자회사)에 100억 물건 판매

A 입장: 매출 100억 발생 ✅
B 입장: 매입 100억 발생 ✅

그런데 연결 재무제표는 A+B를 하나의 회사로 보기 때문에, 이 100억은 "왼손이 오른손에 판 것" 으로 실제 외부 매출이 아닙니다. 그래서 반드시 제거


연결 조정 분개 : 내부거래를 제거하기 위해 별도로 만드는 회계 분개  계열사의 개별 장부를 건드리지 않고, 연결 워크시트 위에서만 수정 분개를 추가하는 방식
수작업으로 하면 계열사 수가 많을수록 조합이 폭발적으로 늘어나는 고난이도 작업


에스컬레이션 : 문제가 해결되지 않을 때 더 높은 권한을 가진 사람에게 자동으로 보고/전달되는 것


총계정원장 : 회사에서 발생한 모든 거래를 계정과목별로 정리한 "회계의 핵심 장부"
